1. Как настроить SSH-ключ для безопасного взаимодействия с GitHub?
Генерируем ключ в Power shell ssh-keygen -o -t rsa -C "@...."
Добавляем ключ в GitHub

2. Как использовать SSH вместо HTTPS для клонирования репозитория?
Клонируйте через SSH:
git clone git@github.com:username/repository.git
Измените URL на SSH:
git remote set-url origin git@github.com:username/repository.git

3. Какие основные разделы и информацию следует включать в файл README?
Название, описание, установка, использование, документация, вклад, лицензия, контакты.

4. Как обновления в файле README могут влиять на управление проектом?
Улучшение документации, привлечение участников, актуальность проекта.

5. Какие ошибки чаще всего встречаются при настройке SSH для GitHub?
Не добавлен ключ в GitHub.
Не запущен SSH-агент.
Неправильные права на файл ключа.
Файервол блокирует порт 22.
